  1. 物件を先に確保
  2. ルームメイトを募集

ルームメイトファースト

  1. 理想のルームメイトを探す
  2. 相性を確認してマッチング
  3. 2人で一緒に物件探し

メリット: 相性の良い人と理想の物件を見つけられる

👥 ターゲットユーザー

主要ターゲット: 大学生

  • 年齢層: 18-25歳
  • 状況:
    • 一人暮らしは家賃が高すぎる
    • 同じ大学・同じ価値観の人と住みたい
    • これから物件を探す段階
    • 信頼できるルームメイトが見つからない
  • ニーズ:
    • 家賃を抑えたい（月3-5万円）
    • 同世代・同じ大学のコミュニティ
    • 生活リズムが合う人
    • 安全・信頼できる相手

セカンダリーターゲット

  • 若手社会人: 20代、転職・上京したばかり
  • 外国人留学生: 日本での生活を始める人
  • フリーランス: コストを抑えたい、コミュニティが欲しい

⚠️ 既存サービスの共通課題

  • 物件ありき: すべて「物件が先、人が後」
  • 相性マッチングなし: 単純な条件フィルターのみ
  • 学生特化なし: 大学コミュニティや学生証認証がない
  • 物件探しサポートなし: マッチング後のサポートがない
  • 信頼性が低い: 誰でも投稿でき、トラブルが多い

1. ユーザー管理

✅ 本人確認書類不要 + 学生証認証（任意）

法的リスクを回避しつつ、学生コミュニティの信頼性を確保

認証システム
  • 必須認証
    • メールアドレス認証
      • 大学メール（.ac.jp）は信頼スコア加点
      • 使い捨てメールアドレスの検出・ブロック
    • SMS認証
      • 1電話番号 = 1アカウント
      • 重複登録防止
  • 任意認証（信頼スコア向上）
    • 学生証認証
      • 学生証の写真アップロード
      • 運営が目視確認（個人情報は見えないよう加工推奨）
      • 認証後「学生認証済み」バッジ表示
      • 信頼スコア +15点
    • SNS連携
      • Google、Twitter/X、Instagram
      • 各連携で信頼スコア +5点
プロフィール管理
  • 基本情報
    • ニックネーム
    • 年齢
    • 性別（男性/女性/その他/非公開）
    • 学生区分（大学生/大学院生/専門学生/社会人/その他）
    • 大学情報（学生の場合）
      • 大学名
      • 学部・学科
      • 学年
      • キャンパス
    • 職業（社会人の場合）
  • 物件探しの希望条件
    • 予算
      • 希望家賃（1人あたり）: 3-5万円 / 5-7万円 / 7万円以上
      • 初期費用の上限
    • エリア
      • 希望地域（都道府県、市区町村）
      • 最寄り駅（複数選択可）
      • 大学までの通学時間
    • 物件タイプ
      • 希望間取り（1LDK / 2DK / 2LDK等）
      • 築年数
      • こだわり条件（バストイレ別、オートロック等）
    • 入居希望時期
      • すぐ / 1-3ヶ月後 / 3-6ヶ月後 / 未定
  • ライフスタイル情報
    • 睡眠時間帯（早寝早起き/夜型/不規則）
    • 喫煙（する/しない/電子タバコのみ）
    • ペット（飼いたい/既に飼っている/苦手/どちらでも）
    • 来客頻度（ほぼない/月1-2回/週1回以上）
    • 在宅時間（ほぼ外出/半々/ほぼ在宅）
    • 清潔度（1-5段階）
    • 騒音許容度（1-5段階）
    • 社交性（1人時間重視/適度に交流/一緒に過ごしたい）
    • 料理頻度（しない/時々/毎日）
  • 趣味・興味
    • タグ形式（料理、スポーツ、読書、映画、音楽、ゲーム、旅行等）
    • 最大10個まで
  • 自己紹介
    • テキスト（500文字）
    • 自己紹介動画（30秒、任意）
    • プロフィール写真（推奨）
信頼スコアシステム（0-100点）
項目 配点 条件
プロフィール完成度 20点 すべての項目を入力
メール認証 10点 メール認証完了
大学メール +5点 .ac.jpドメイン
SMS認証 15点 電話番号認証完了
学生証認証 15点 学生証アップロード・承認
SNS連携 各5点（最大15点） Google/Twitter/Instagram
レビュー評価 20点 平均評価×4点
アクティビティ 10点 レスポンス速度、メッセージの質
利用期間 10点 長期利用で加点

2. スマートマッチング

ライフスタイル診断（20問）

相性を科学的に分析し、トラブルを事前に予防

  • 生活リズム（5問）
    • 平日の起床時間
    • 平日の就寝時間
    • 休日の過ごし方
    • 夜中の活動（勉強、ゲーム等）
    • 朝のバスルーム使用時間
  • 清潔度・片付け（5問）
    • 掃除の頻度
    • 共有スペースの使い方
    • 食器洗いのタイミング
    • ゴミ出しのルール
    • 散らかり許容度
  • 社交性・コミュニケーション（5問）
    • 友人を呼ぶ頻度
    • ルームメイトとの交流希望度
    • 一緒に食事したい頻度
    • 騒音の許容度
    • プライバシーの重要度
  • 金銭感覚・ルール（5問）
    • 共有費用の考え方
    • 家賃の支払い方法
    • 節約志向度
    • ルールの厳格度
    • トラブル時の対応方法
相性スコア表示
  • 総合相性スコア（0-100点）
    • 90-100点: 理想的なマッチ（超おすすめ）
    • 75-89点: 高相性（おすすめ）
    • 60-74点: 良好（試してみる価値あり）
    • 40-59点: 要検討（よく話し合いが必要）
    • 0-39点: 低相性（おすすめしない）
  • カテゴリ別相性
    • 生活リズム相性
    • 清潔度相性
    • 社交性相性
    • 金銭感覚相性
    • 趣味・興味の一致度
  • 相性の理由
    • 「2人とも早起きで生活リズムが合います」
    • 「清潔度の感覚が近いです」
    • 「プライバシーを尊重し合えそうです」
  • 注意点
    • 「来客頻度に差があります。事前に話し合いましょう」
    • 「騒音の許容度が異なります」
検索・フィルター機能
  • 大学・学生関連
    • 同じ大学
    • 同じキャンパス
    • 同じ学年
    • 同じ学部・近い学部
    • 学生認証済みのみ
  • 希望エリア
    • 都道府県、市区町村
    • 最寄り駅
    • 大学までの通学時間
  • 予算
    • 希望家賃の範囲が近い
    • 初期費用の予算
  • 入居時期
    • すぐ / 1-3ヶ月 / 3-6ヶ月 / 未定
  • ライフスタイル
    • 喫煙の可否
    • ペットの可否
    • 性別
    • 年齢層
    • 生活リズム
    • 清潔度レベル
    • 社交性
  • 表示順
    • 相性スコア順（デフォルト）
    • 同じ大学優先
    • 新着順
    • 信頼スコアが高い順
    • 予算が近い順
AIレコメンデーション
  • ユーザーの閲覧履歴から学習
  • 相性の高い候補を自動提案
  • 「同じ〇〇大学の人が見つかりました」通知
  • 週1回のおすすめメール配信

🗄️ データベース設計

設計方針

  • ルームメイトファースト: ユーザー間のマッチングが中心
  • 物件は副次的: 物件情報は「探している物件の希望条件」として保存
  • 物件候補リスト: マッチング後に2人で共有する物件リスト機能
-- ユーザーテーブル
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    phone_number VARCHAR(20) UNIQUE,
    phone_verified BOOLEAN DEFAULT FALSE,
    email_verified BOOLEAN DEFAULT FALSE,
    is_university_email BOOLEAN DEFAULT FALSE, -- .ac.jp等
    trust_score INTEGER DEFAULT 0 CHECK (trust_score >= 0 AND trust_score <= 100),
    status VARCHAR(20) DEFAULT 'active',
    last_login_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- プロフィールテーブル
CREATE TABLE profiles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    nickname VARCHAR(50) NOT NULL,
    age INTEGER,
    gender VARCHAR(20),
    user_type VARCHAR(20), -- student, graduate, working, other

    -- 学生情報
    university_name VARCHAR(200),
    faculty VARCHAR(200),
    department VARCHAR(200),
    grade INTEGER, -- 学年
    campus VARCHAR(200),

    -- 社会人情報
    occupation VARCHAR(100),

    bio TEXT,
    video_url VARCHAR(500),
    avatar_url VARCHAR(500),

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id)
);

-- 学生証認証テーブル
CREATE TABLE student_verifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    student_card_image_url VARCHAR(500), -- 加工済み画像
    university_name VARCHAR(200),
    status VARCHAR(20) DEFAULT 'pending', -- pending, approved, rejected
    verified_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id)
);

-- SNS連携
CREATE TABLE social_connections (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    provider VARCHAR(50) NOT NULL,
    provider_user_id VARCHAR(255) NOT NULL,
    connected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(provider, provider_user_id)
);

-- 物件希望条件テーブル（ユーザーが探している物件の条件）
CREATE TABLE property_preferences (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,

    -- 予算
    min_rent DECIMAL(10, 2),
    max_rent DECIMAL(10, 2),
    max_initial_cost DECIMAL(10, 2),

    -- エリア
    preferred_prefectures TEXT[], -- 配列
    preferred_cities TEXT[],
    preferred_stations TEXT[],
    max_commute_time INTEGER, -- 分

    -- 物件タイプ
    room_types TEXT[], -- 1LDK, 2DK, etc.
    min_building_age INTEGER,
    max_building_age INTEGER,

    -- こだわり条件
    requires_auto_lock BOOLEAN DEFAULT FALSE,
    requires_separate_bath_toilet BOOLEAN DEFAULT FALSE,
    requires_south_facing BOOLEAN DEFAULT FALSE,
    other_requirements TEXT,

    -- 入居時期
    move_in_timing VARCHAR(50), -- immediate, 1-3months, 3-6months, undecided

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id)
);

-- ライフスタイル
CREATE TABLE lifestyles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    sleep_schedule VARCHAR(20),
    cleanliness_level INTEGER CHECK (cleanliness_level >= 1 AND cleanliness_level <= 5),
    smoking BOOLEAN DEFAULT FALSE,
    e_cigarette BOOLEAN DEFAULT FALSE,
    pets VARCHAR(20), -- have, want, allergic, neutral
    guest_frequency VARCHAR(20),
    noise_tolerance INTEGER CHECK (noise_tolerance >= 1 AND noise_tolerance <= 5),
    socializing_preference INTEGER CHECK (socializing_preference >= 1 AND socializing_preference <= 5),
    home_time VARCHAR(20), -- mostly_out, half, mostly_home
    cooking_frequency VARCHAR(20), -- never, sometimes, daily
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id)
);

-- 興味・趣味
CREATE TABLE interest_tags (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(50) UNIQUE NOT NULL,
    category VARCHAR(50)
);

CREATE TABLE user_interests (
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    interest_tag_id UUID REFERENCES interest_tags(id) ON DELETE CASCADE,
    PRIMARY KEY (user_id, interest_tag_id)
);

-- ライフスタイル診断結果
CREATE TABLE lifestyle_assessments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    question_id INTEGER NOT NULL,
    answer_value INTEGER NOT NULL,
    completed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, question_id)
);

-- マッチングリクエスト
CREATE TABLE match_requests (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sender_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    receiver_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    status VARCHAR(20) DEFAULT 'pending', -- pending, accepted, rejected, expired
    message TEXT,
    compatibility_score INTEGER CHECK (compatibility_score >= 0 AND compatibility_score <= 100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP,
    CHECK (sender_user_id != receiver_user_id)
);

-- マッチングペア（承認後のペア）
CREATE TABLE matched_pairs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user1_id UUID REFERENCES users(id) ON DELETE CASCADE,
    user2_id UUID REFERENCES users(id) ON DELETE CASCADE,
    match_request_id UUID REFERENCES match_requests(id),
    status VARCHAR(20) DEFAULT 'active', -- active, property_searching, contracted, dissolved
    matched_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    dissolved_at TIMESTAMP,
    CHECK (user1_id < user2_id), -- 重複防止
    UNIQUE(user1_id, user2_id)
);

-- 物件候補リスト（マッチング後に2人で共有）
CREATE TABLE property_candidates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    matched_pair_id UUID REFERENCES matched_pairs(id) ON DELETE CASCADE,

    -- 物件情報（外部サイトから取得またはユーザー入力）
    source_url VARCHAR(500), -- SUUMO等のURL
    title VARCHAR(200),
    address TEXT,
    rent DECIMAL(10, 2),
    initial_cost DECIMAL(10, 2),
    room_type VARCHAR(20),
    building_age INTEGER,
    station_name VARCHAR(100),
    station_walk_minutes INTEGER,

    image_urls TEXT[], -- 画像URL配列
    description TEXT,

    -- 2人の評価
    user1_rating INTEGER CHECK (user1_rating >= 1 AND user1_rating <= 5),
    user2_rating INTEGER CHECK (user2_rating >= 1 AND user2_rating <= 5),
    user1_comment TEXT,
    user2_comment TEXT,

    -- ステータス
    status VARCHAR(20) DEFAULT 'candidate', -- candidate, favorite, viewing_scheduled, applied, contracted, rejected

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 共有ウィッシュリスト（ペアの共通希望条件）
CREATE TABLE pair_preferences (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    matched_pair_id UUID REFERENCES matched_pairs(id) ON DELETE CASCADE,

    agreed_min_rent DECIMAL(10, 2),
    agreed_max_rent DECIMAL(10, 2),
    agreed_max_initial_cost DECIMAL(10, 2),
    agreed_areas TEXT[],
    agreed_stations TEXT[],
    agreed_room_types TEXT[],
    priority_order TEXT[], -- 優先順位

    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(matched_pair_id)
);

-- 内見スケジュール
CREATE TABLE viewing_schedules (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    property_candidate_id UUID REFERENCES property_candidates(id) ON DELETE CASCADE,
    scheduled_date TIMESTAMP NOT NULL,
    location TEXT,
    notes TEXT,
    status VARCHAR(20) DEFAULT 'scheduled', -- scheduled, completed, cancelled
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- メッセージ
CREATE TABLE messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sender_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    receiver_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    matched_pair_id UUID REFERENCES matched_pairs(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    is_flagged BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    read_at TIMESTAMP,
    CHECK (sender_user_id != receiver_user_id)
);

CREATE INDEX idx_messages_pair ON messages(matched_pair_id, created_at DESC);

-- レビュー
CREATE TABLE reviews (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    reviewer_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    reviewee_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    matched_pair_id UUID REFERENCES matched_pairs(id) ON DELETE CASCADE,

    cleanliness_rating INTEGER CHECK (cleanliness_rating >= 1 AND cleanliness_rating <= 5),
    communication_rating INTEGER CHECK (communication_rating >= 1 AND communication_rating <= 5),
    rule_compliance_rating INTEGER CHECK (rule_compliance_rating >= 1 AND rule_compliance_rating <= 5),
    financial_reliability_rating INTEGER CHECK (financial_reliability_rating >= 1 AND financial_reliability_rating <= 5),
    overall_rating INTEGER CHECK (overall_rating >= 1 AND overall_rating <= 5),

    positive_comment TEXT,
    improvement_comment TEXT,
    advice_for_next TEXT,

    visibility VARCHAR(20) DEFAULT 'public',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CHECK (reviewer_user_id != reviewee_user_id),
    UNIQUE(reviewer_user_id, reviewee_user_id, matched_pair_id)
);

-- 通報
CREATE TABLE reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    reporter_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    reported_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    reason VARCHAR(100) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    resolved_at TIMESTAMP
);

-- ブロックリスト
CREATE TABLE blocked_users (
    blocker_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    blocked_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    blocked_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (blocker_user_id, blocked_user_id)
);

-- お気に入りユーザー
CREATE TABLE favorite_users (
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    favorite_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, favorite_user_id)
);

-- 閲覧履歴
CREATE TABLE view_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    viewed_user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_view_history ON view_history(user_id, viewed_at DESC);

-- 通知
CREATE TABLE notifications (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    type VARCHAR(50) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    link_url VARCHAR(500),
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    read_at TIMESTAMP
);

CREATE INDEX idx_notifications_user ON notifications(user_id, is_read, created_at DESC);

-- 大学コミュニティ投稿
CREATE TABLE university_posts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    university_name VARCHAR(200) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    category VARCHAR(50), -- question, advice, property_info, event
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 大学コミュニティコメント
CREATE TABLE university_post_comments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    post_id UUID REFERENCES university_posts(id) ON DELETE CASCADE,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- インデックス
CREATE INDEX idx_users_trust_score ON users(trust_score DESC);
CREATE INDEX idx_profiles_university ON profiles(university_name, campus);
CREATE INDEX idx_match_requests_status ON match_requests(status, created_at DESC);
CREATE INDEX idx_property_candidates_pair ON property_candidates(matched_pair_id, status);

📄 まとめ

🎯 このサービスの革新性

「人が先、物件は後」という既存サービスとは真逆のアプローチ

なぜこれが革新的なのか？

  • ✅ 相性重視: 物件ありきだと妥協が必要。人ありきなら理想を追求できる
  • ✅ 学生ニーズに最適: 予算が限られる学生こそ、相性の良い相手と効率的に探したい
  • ✅ トラブル予防: 相性診断で事前にミスマッチを防ぐ
  • ✅ コミュニティ形成: 大学別コミュニティで安心感
  • ✅ 物件探しサポート: マッチング後も継続的に価値提供

ユーザージャーニー

  1. 登録 → ライフスタイル診断
  2. 理想のルームメイト検索（相性スコア表示）
  3. マッチング成立
  4. 2人で希望条件をすり合わせ
  5. 物件候補を一緒に探す・評価
  6. 内見 → 契約
  7. 入居後もサポート継続

結果: ユーザーは妥協なく、理想のルームメイトと理想の物件を見つけられます。
